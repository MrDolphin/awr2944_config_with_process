# TI AWR2944 毫米波雷达项目性能优化全纪实

> **项目背景**：基于 TI AWR2944P 的毫米波雷达配置与实时处理系统，原始版本在 200ms (5Hz) 下存在严重数据延迟、目标跟踪极易丢失以及多普勒计算精度不足等问题。

---

## 🛠️ 第一阶段：核心算法与物理公式修正 (Analysis & Tuning)

### 1. 测速上限瓶颈修复 (Bug Fix)
*   **定位问题**：原始代码在计算 `maxVelocity` (最大不模糊速度) 时，错误地除以了 `chirpsPerFrame`。
*   **影响**：导致雷达识别的运动上限仅为 `0.08m/s`，超过此速度目标即刻丢失。
*   **修复**：校准公式为 `maxVelocity = 3e8 / (4 * startFreq * 1e9 * chirpDuration)`。
*   **结果**：量程恢复至真实的 **~9.74m/s**。

### 2. 刷新率飞跃：从 5Hz 到 20Hz (Performance)
*   **改动**：将 MATLAB 内部定时器周期 `Period` 从 `200ms` 直接下调至 **`50ms`**。
*   **效果**：彻底消除了 UDP 数据包在后台排队产生的“视野时间差”，让可视化变得即时、顺滑。

### 3. 目标跟踪稳定性增强 (Robustness)
*   **低反射建轨**：`pointsThre` (初始化点数门限) 由 `5` 降至 **`2`**，解决了侧身时因点数不足导致的“视而不见”。
*   **长久记忆力**：`act2freeThre` (轨迹丢失门限) 由 `3` 猛增至 **`20`** (约 1 秒容错)，解决了目标快速移动或短暂遮挡导致的轨迹闪烁。

## 🚣‍♀️ 第四阶段：从室内模型到外场实战的架构大重构 (v2.0 Architecture Transition)
> **日期: 2026-03-23**
> **核心目标**: 彻底排查前端轨迹卡死飘移问题，抛弃不可靠的硬件追踪，自研软件解算层，并打通底层离线数据重演链路。

### 1. 轨迹真理之战：前端全自研 DBSCAN (Tracker Rewrite)
*   **问题**：早期版本强依赖雷达硬件吐出的 Type 7 (Target List)，但 AWR2944 该固件下输出的其实是驻留区域中心（坐标长期为 0），导致海量幽灵目标重叠。
*   **重构**：在 `radar_app.html` 中彻底切断硬件追踪，白手起家手写了一套 **JavaScript DBSCAN 聚类与连续多目标追踪器 (TargetTracker)**，实现了每一个目标的平滑追踪和独立颜色连贯航线绘制。
*   **盲点与修复**：为了消除室内极其恶劣的墙壁“多径镜像”，曾一度将聚类半径拉大到 `3.5m`，结果导致真实人体和镜像被平均计算，航迹死死卡在房间中央。现已精准调回正常船体尺度的 **`1.2m`**，配合 `> 0.05m/s` 的微小多普勒过滤，完美解封了大范围画圆轨迹，并在空房间内实现零误报。

### 2. 后端基建：彻底终结“串口僵尸死锁” (Backend Deadlock Fix)
*   **问题**：每次按下 Ctrl+C 强杀程序并重新运行时，`radar_server.py` 都会莫名其妙长睡不醒（假死），必须退出再重进才能苟活。
*   **根源**：强杀时雷达仍在以 3Mbps 速率向 Linux 内核灌数据，重开后 Python 读到了截断的旧包遗骸，将乱码解析成了动辄几个 GB 的包长，从而陷入了“无限等待数据补齐”的跨世纪单相思死锁。
*   **神级补丁**：
    1. 连上串口第一秒，立刻执行 `data_port.reset_input_buffer()` 核弹级清空旧账。
    2. 加入 `total_packet_len < 40 or > 1000000` 长度变态锁，一旦触发直接手起刀落切掉废弃的魔法字强迫从头寻找，脚本自此如丝般顺滑，再无卡顿。

### 3. 终极分析兵器：底层二进制解析与录制全家桶 (Offline Data Pipeline)
*   **痛点**：网页前端的数据为了性能丢弃了多维特征，无法用于算法二次验证；而原厂的 `load_radar_data.m` 只能读截获卡的 ADC，读不了串口的点云。
*   **双管齐下**：
    1. **Python 落盘引擎**：为 `radar_server.py` 加上了 `--log_file` 参数。若是 `.jsonl` 则无损保留全浮点 $(X,Y,Z,V)$，若是 `.bin` 则原封不动地暴写 UART Binary 原生字节流。
    2. **MATLAB 解包脚本**：我在本地专为您量身定做了 `parse_uart_bin.m` 脚本！它能在 MATLAB 中秒级完成魔法字同步、解帧、以及把点云的坐标和速度重组为了结构体，甚至还带上了一个绝美的动态 3D 播放器彩蛋，彻底打通了算法重演的任督二脉。

### 4. 噪音清洗：CFAR 阈值与 Debug 矩阵 (Hardware DSP Tuning)
*   **问题**：没人的时候空房间里竟然有 3 个稳如泰山的幽灵目标。这是因为之前的速度阈值设为 0，导致反光极强的死物被录入。
*   **硬件制裁**：修改了 `profile-2944.cfg`：
    1. 把 `cfarCfg` 阈值系数一举从 `15` 猛抬到 `18`，从纯物理层暴力砍断了室内多径散射中 90% 的虚弱回波。
    2. 开启了 `guiMonitor -1 1 1 0 0 1 1`，现已将一维距离图 (Range Profile) 和顶级二维距离多普勒热力图 (Range-Doppler Heatmap) 源源不断地压入 `.bin` 存档，为离线算法排雷提供了降维打击的分析材料。
